Archlinux更新软件，出现与curl有关的错误的解决办法

发布 2019, Dec 11 需要1分钟读完

今天下载软件的时候，因为网络问题，速度一直为0，就ctrl C 强行停止了，

之后再继续更新的时候就报错，与 curl 有关，如下

HTTP server doesn't seem to support byte ranges. Cannot resume.

查了一圈之后发现也有其他人遇到这个问题，

原因就是这个源不支持断点续传，解决办法最好是清楚缓存，

如果是用 pacman 过程中发生的错误，那么就运行

sudo pacman -Scc

如果是用 yay 过程中发生的错误，那么运行

yay -Scc

之后输入几个 y 就可以了

之后继续更新或者下载之前的软件
