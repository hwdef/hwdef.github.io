用时间换才华

我对一些linux发行版的印象

发布 2019, Nov 08 需要4分钟读完

从高一开始第一次接触 linux，我也多多少少装过几十个 linux 的发行版了，其中有的毫无特点，装完就格式化了，有些让我感到惊艳，也有一些发行版，通过各种方式折磨我，由此在我心中留下了深深的烙印。

其实我一直算是一个软粉，第一部属于我自己的手机是 nokia 5233， 第二部就是 lumia 920。

但是我依然对 linux 抱有极大的热爱，我希望能更早遇到 linux 就好了。但我觉得无论是 windows，还是 linux 都是非常优秀的操作系统。

也正因为我对这两个操作系统相同的喜爱程度，导致大多数时间，我都是使用双系统，这也影响了我的学习和工作的效率。

由于我目前从事 kubernetes develop 相关的工作，可能以后我会更多的使用 linux 吧，现在阻挡我只使用 linux 的原因是 office。

这时候可能有人会推荐 mac os，其实直言不讳的说，我一直都是个果黑，随着时间的前进，我对苹果这家公司的厌恶程度也愈发加深。原因很多，简单总结的话，前期厌恶是因为苹果的用户，后期厌恶是因为苹果的产品。

那么接下来进入正题，我想说一说一些 linux 发行版在我心中的印象。

ubuntu:

我对 ubuntu 的评价只有“平庸”二字，

不过无可厚非，Canonical 作为一家商业公司，为 linux 的普及做了很多贡献，但是也无法掩盖它在 linux 社区贡献很少的表现。

ubuntu 的设计美学很差，用户界面一代不如一代，本来 gnome 就很丑了，但仍会给人一种“坚实”的感觉，ubuntu 魔改过的 gnome 更加不堪入目，之前的 unity 还算有点特色，可惜为了省钱，还是被砍了。

我觉得 ubuntu 还是有一个优点的，就是n卡独显的驱动比较好装，装上直接就能用，不像其他 linux ，装不好就崩了。

除此之外，味如嚼蜡。

archlinux:

archlinux 是我最喜欢的 linux 发行版，无论是从名字，logo，还是对于“精简”这个词的理解上，archlinux 做的都是最好的。

它会给人一种，我的系统是我自己从零创造的错觉。这种满足感，正是程序员需要的。

archlinux 的 wiki 也称得上是 linux 届的百科全书，无论你用的什么发行版，archlinux wiki 都可以对你有所帮助，它甚至会帮助你理解整个 linux。

fedora:

fedroa 是我目前正在使用的发行版，激进与稳定的结合。适合用来工作。

debian/centos:

这两个我觉得可以放在一起说，他们虽然很多地方不同，但是发展思路是相同的。他们适合做服务器系统。

很稳定，很优秀。

manjaro:

manjaro 是后起之秀，它站在巨人 archlinux 的肩膀上，

优化了安装流程，优化了桌面程序，优化了驱动安装

但是我不是很喜欢，

安装流程很多坑，桌面程序很多坑，驱动安装也很多坑

既然做出来一个功能，那么就应该保证它的可用性，否则就别做。

可能它需要时间，但现在它很糟。

kubuntu:

kde + ubuntu

在我的印象里只有崩溃，安装程序都崩溃，不知道现在是否得到改善。

ubuntu kylin:

特供版ubuntu

别用。

tails:

据说是最安全的 linux 版本，

它做好了两个方面，

一是默认使用tor网络，但是在国内要先翻墙，tor才能正常用。

二是使用 linux 的 live cd 特性，关机后不会留下痕迹。

puppy:

为小而生的 linux 发行版，特殊之处是适合u盘，用起来没什么其他的感觉

slax:

同样是适合u盘的系统，我喜欢。

应用程序的安装很方便，

界面很好看。不过现在的界面不好看了，我用的时候 slax 默认用的 kde4，我对 kde 的了解就是从这开始的，现在的界面还如之前。

kali:

被誉为是黑客专用系统，emm不过怎么说呢。。。我是不相信真正的黑客会用这个系统的。。

其实你用任何一个 linux 都可以自己做出 kali linux。

官网支持的平台特别多，这点做的很好。

deepin:

好看倒是好看，但是也太像 mac os 了，拒绝。

qubes:

了解了一下感觉很厉害，但是从来没安装成功过，不好评价。logo倒是不丑。

opensuse:

opensuse 的设计美学我是认可的，驱动安装也算方便，

包管理器在国内的使用体验一般，

对新手很友好，可以替代 ubuntu。

我喜欢 kde 桌面，opensuse 和 kde 的融合做的很好，可能是因为都在德国开发，同一批开发人员，能更好更快的解决问题

gentoo:

没装过，了解到需要编译安装包，感觉有点浪费时间。
