Github下fork后如何同步源的新更新内容

发布 2019, Jan 09 需要1分钟读完

本文是整理自知乎回答，

gitlab或github下fork后如何同步源的新更新内容？

就当做个笔记，防止自己忘记。

1.查看远端状态

git remote -v

2.添加远端

git remote add upstream <远端仓库地址>

3.再次查看远端状态，检查远端是否添加成功

git remote -v

4.从上游仓库 fetch 分支和提交点，提交给本地 master，并会被存储在一个本地分支 upstream/master

git fetch upstream

5.切换到本地分支

git checkout master

6.把 upstream/master 分支合并到本地 master 上

git rebase upstream/master

7.推送到自己的github上

git push origin master
